How to move wordpress from localhost to production server.

the easy way

Wondering how to move your local wordpress installation to an online production server that is your selected web hosting partner?. Its quite easy.

If you haven't yet started a wordpress blog, you can read our beginner tutorial to starting a wordpress blog here.

Basic steps to move wordpress from localhost to production server.

  1. Access your localhost wordpress installation.
  2. Download a copy of all the files in the installation.
  3. Make an export of your database.
  4. Access the file manager of your online host, either through ftp or host cpanel.
  5. Upload the wordpress files you made from localhost.
  6. Import your localhost database through phpmyadmin.
  7. Edit both your wp-config file and your database wp-options.
  8. All done!

Access your localhost wordpress installation.

You need to gain access to your wordpress site on your localhost machine.

On most machines, it will be

http://localhost/wordpress

If you use Xampp to run your wordpress site, follow the following steps:

  • Launch xampp control panel
  • Navigate to localhost on your web browser.
  • On the top right menu, click applications and choose access wordpress
  • Navigate to wp-admin and login to your dashboard.

I would advise you to change your password here and set your preferred password you plan to use on your production server.

To do this, go to edit my profile on top right bar and change the password.

Download a copy of all the files in the installation.

Next you need to make a copy of the whole wordpress installation. I would recommend compressing it into a zip format for an easy upload.

In windows xampp environment, the wordpress files can be found on the xampp directory inside the apps folder. That is /xampp/apps/wordpress/htdocs/

If you are using the default php-apache on linux, the files can be found at /var/www/html/wordpress/ folder.

While if you are on archlinux the files are located at /srv/apache/wordpress/ folder.

As i had said, compress everything inside the wordpress folder and convert it into a zip file.

Make an export of your database.

You need to export your database as it contains all the posts and pages contained in your wordpress blog. Its the core of your site.

I will assume you already have phpmyadmin installed on your machine.

Its usually already included on the xampp environment. If on linux use the command below to install it

sudo apt-get install phpmyadmin

or if on archlinux

sudo pacman -Syuu phpmyadmin

On xampp control panel, start mysql and navigate to localhost/phpmyadmin

Select your wordpress database on the left column and expand it.

Next click on export option from the top bar. This will export your wordpress database to preferred location on your local machine.

Access the file manager of your online host, either through ftp or host cpanel.

Most hosting providers have an inbuilt file manager on the control panel. I would recommend an ftp since most of these file managers come with upload size restriction.

The downloaded zip file from your localhost environment is most likely to be huge depending on the size and content of your blog. That makes it hard to upload via the file manager.

However, you can upload the individual files through it thou this is a complete long process since wordpress contains thousands of files. Resort to this method as the last option, that is incase all other methods fail.

For ease, use an ftp tool example filezilla or duckzilla. Download filezilla here incase you haven't already.

Use your ftp details provided by your hosting provider to log in to your account and upload the localhost wordpress zip file to your root directory or a folder if you plan to.

Unzip the file while in the root directory. Most ftp programs and also the inbuilt file managers provided have this option so it shouldn't be a problem.

You can also use a php script available at github which auto unzips your files. Its efficient and fast. Download the script using this link. Upload the unzipper script to your root directory and grant it read, write and execute permissions.

Navigate to yoursite.com/unzipper.php and follow the onscreen prompts to extract your file.

Import your localhost database through phpmyadmin.

Access your web hosting phpmyadmin. Create a new database which will be used with your wordpress blog. Also create a new user and grant him all priviledges to use the database.

In some cases, the web hosting provider usually has already created a database for you to use. The database details are usually available on your control panel.

Note down the following:

  • The database name
  • Database server or host
  • Your database username
  • Finally your username password

Navigate to import option on your top bar.

An option to upload a database will be provided. This is where the database we exported from the localhost environment comes in.

Select the database and ignore all the other options provided on the page. Select go and your old localhost database will be imported to the new database you just created.

Edit your phpmyadmin database wp-options

Using your phpmyadmin, login and access your database. You will need your database server, username and password to login. Most web hosting providers have phpmyadmin inbuilt in their control panel so you wont need to login.

On the left column, select your database if you have more than one databases. If you only have one, its already autoselected.

Look for the row marked as wp-options and select it. This opens your wordpress options. Here you can also retrieve your site incase you are unable to access your admin dashboard.

Edit the site url option and change it from your localhost address to your new domain name. That is from http://localhost/wordpress to http://yoursite.com.

Edit your home url option to the same address as above.

You can also change your admin email via the admin_email row and also your blogname via the blogname option and description via the blogdescription row.

Save your configurations.

Edit both your wp-config file and your database wp-options.

The final step would be to edit your wp-config and database wp-options.

Wp-config file is located inside your wordpress folder.

You need to change the database information from the one you used on your localhost machine to the new one you just created.

If you don't have the wp-config file , just create one and add the following:

// ** MySQL settings - You can get this info from your web host ** //
/** The name of the database for WordPress */
define( 'DB_NAME', 'bitnami_wordpress' );

/** MySQL database username */
define( 'DB_USER', 'myuser' );

/** MySQL database password */
define( 'DB_PASSWORD', 'mypass123' );

/** MySQL hostname */
define( 'DB_HOST', 'localhost:3306' );

/** Database Charset to use in creating database tables. */
define( 'DB_CHARSET', 'utf8' );

/** The Database Collate type. Don't change this if in doubt. */
define( 'DB_COLLATE', '' );

Edit and change:

  • bitnami_wordpress to your new database name
  • myuser to your new database username.
  • mypass123 to the password you set on yout new database
  • localhost:3306 to your web hosting database server. In most cases its just loaclhost but you need to check.
  • The utf8 should remain as it is.

That should be all. You should now be able to access your wordpress site and login to your dashboard using the username and password you previously used on the localhost environment.

You can change the password after logging in.

Use of migration plugins

You can also use automated wordpress plugins to make yout migration process easy.

The recommended plugins are:

  • Duplicator
  • All-in-one Wordpress migration plugin

Duplicator

Duplicator creates a copy of your wordpress themes, plugins, settings and database and creates a zip file.

You don't have to create the backup manually.

Download duplicator here.

All-in-one Wordpress migration plugin

All-in-one Wordpress migration plugin can be able to manuever through any hosting limitations to give you the backup you need.

You get to choose what to carry and what to leave behind when exporting a backup.

Download All-in-one Wordpress migration plugin here.
